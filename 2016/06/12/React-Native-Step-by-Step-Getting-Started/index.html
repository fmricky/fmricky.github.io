React Native Step by Step（一）初识React Native

发表于 2016-06-12   |   分类于 React Native Step by Step   |  

因为公司项目需要，开始研究 Hybrid 技术，进而发现了 Facebook 的 React Native。严格来说，这并不算是传统意义的 Hybrid 技术。它不在WebView里执行，而是基于JavascriptCore，直接执行服务端的JS代码。

引用一位知乎匿名用户的总结

在React Native中，写JS的工程师解决的是「将基本组件拼装成可用的React组件」的问题，写Native Code的工程师解决的是「提供核心组件，提供足够的扩展性、灵活性和性能」的问题

目前只是初学，还无法评论上述总结是否贴切，一步一步学下去吧。笔记里只涉及React Native for iOS。

React vs React Native

对我来说，仅有的一丢丢js知识，基本是基于jQuery的操作，看过的js的书也仅有一本《锋利的jQuery》。css也仅存于css2那个阶段用来简单布局的阶段。工作以来几乎完全没有再在接触js开发。突然接手学React Native，好多基础概念都还是懵懂的。就像譬如说React Native，跟React.js有神马关系？

似乎网上对这个的解释并不多，今天看书时才放到。也许对前端的同学来说，这个可能是常识了。就当给我一样的小白们先科普一下。

最初先冒出来的是React这东西，它是Facebook开源的一个前端项目，用官方的解释是A javascript library for building user interfaces。它的开源项目地址是http://facebook.github.io/react/.

React Native 是基于React的理念用于开发Native App的开源工具。官方对它的描述是A framework for building native APPs using React.这说明React Native采用的是React的语法。同时React的界面构建利用了JSX技术，后续我会详细利用一篇来描述这部分内容。同样提供一下React Native的开源项目地址http://facebook.github.io/react-native/.
